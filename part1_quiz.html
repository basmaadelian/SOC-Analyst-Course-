Answers & Explanations

Read every explanation carefully — understanding why an answer is correct is more valuable than the score.

Q01 An unauthorized employee reads patient records without changing anything.
✓ Correct Answer: B — Confidentiality
Confidentiality means information is accessible only to authorized parties. The unauthorized employee reading the records is a confidentiality violation — even though nothing was changed or deleted. No modification occurred, so integrity was not violated. The data was still accessible, so availability was not violated. This is a classic access control failure: the person was not authorized to see that data, but could. In a SOC context, this scenario would be detected through access logs showing an account querying records outside its role.
Q02 Ransomware encrypts all files, making them inaccessible.
✓ Correct Answer: C — Availability
Ransomware is primarily an availability attack. The files are not stolen (confidentiality is not necessarily violated) and they are not modified in a meaningful way (integrity is not the primary concern). The damage is that authorized users can no longer access their own data. This is why ransomware is so devastating in healthcare — hospitals cannot access patient records, medications lists, or lab results. Restoring availability requires either paying the ransom or recovering from clean, offline backups.
Q03 An attacker changes a bank transfer's destination account number in transit.
✓ Correct Answer: B — Integrity
Integrity means data has not been modified by unauthorized parties. The attacker changed the data — the account number — while it was in transit. The data arrived at the bank, so availability was not violated. The bank could read the data, so confidentiality was not the issue. The problem is that the data was altered. This is a man-in-the-middle attack against integrity. TLS prevents this by providing both encryption (confidentiality) and integrity verification — if the data changes in transit, the integrity check fails and the connection is rejected.
Q04 Salary data accidentally published through a public API endpoint.
✓ Correct Answer: B — Data Loss Prevention (DLP)
A DLP (Data Loss Prevention) solution monitors for sensitive data leaving the organization in unauthorized ways. It can scan outbound traffic and API responses for patterns matching confidential data — such as salary figures, employee IDs, or personal information — and block or alert on those transfers. FIM monitors file changes on disk, not API traffic. IDS looks for attack signatures, not data classification violations. A firewall rule blocks ports and IPs, not specific data patterns. DLP is the control built specifically for this scenario.
Q05 FIM alert: /etc/passwd modified at 02:30 AM Sunday with no change ticket.
✓ Correct Answer: B — An integrity attack, possibly an attacker establishing persistence
/etc/passwd is the Linux file that stores user account information. Modifying it outside an approved change window — especially at 02:30 AM on a weekend — is a major red flag. Attackers who gain access to a Linux system often add new accounts or modify existing ones in /etc/passwd to maintain persistent access even if their original entry point is closed. This is an integrity signal: a critical system file was changed without authorization. It is not a routine update (those require change tickets and rarely happen at 02:30 AM). As an analyst, this requires immediate investigation.
Q06 Aggressive MFA locks out analysts during a breach because the MFA server is down.
✓ Correct Answer: B — Confidentiality vs. Availability
This is a classic Confidentiality vs. Availability conflict. MFA was added to strengthen confidentiality — ensuring only authorized users can access critical systems. But by making MFA mandatory for emergency response tools without a failover plan, the organization accidentally broke availability during an incident. Security engineers must plan for this tension. Emergency access procedures (sometimes called "break glass" accounts) exist for exactly this reason: they allow authorized responders to bypass standard controls during genuine emergencies while maintaining accountability through enhanced logging.
Q07 Classification for an employee's national ID number in the HR system.
✓ Correct Answer: D — Restricted
A national ID number (like a Social Security number or national identity number) is Restricted — the highest classification level. Unauthorized exposure carries severe legal and operational consequences, including identity theft and regulatory penalties. Restricted data includes: government-issued identification numbers, authentication credentials, medical records, and financial account numbers. The classification level determines the protections applied: encryption both at rest and in transit, strict access controls, minimal sharing, and detailed audit logging of who accessed it and when.
Q08 Verifying a software update has not been tampered with during transit.
✓ Correct Answer: B — Compare the SHA-256 hash against the vendor's published hash
Hashing is the correct mechanism for integrity verification. The vendor publishes the SHA-256 hash of their file. After downloading, you compute the hash of your copy and compare. If even a single byte was changed — whether by tampering or corruption — the hash will be completely different. AES-256 is an encryption algorithm, not an integrity check; encrypting a tampered file just produces an encrypted tampered file. File size is a very weak check — an attacker can craft a tampered file of identical size. Antivirus checks for known malware signatures but cannot detect novel tampering. SHA-256 hash comparison is the industry-standard integrity check for software distribution.
Q09 Login with password + one-time code sent to phone.
✓ Correct Answer: C — Authentication
Authentication answers the question "who are you?" Both the password and the one-time code are authentication factors. This is Multi-Factor Authentication (MFA) combining something you know (password) with something you have (the phone receiving the code). Authorization comes after — deciding what the authenticated user is allowed to do. Accounting records what the user actually did. Availability is a CIA pillar, not an AAA component. MFA matters because credential theft (stealing just the password) is one of the most common attack vectors — MFA makes stolen passwords much less useful.
Q10 After login, the system checks if the user can open the finance database.
✓ Correct Answer: B — Authorization
Authorization answers the question "what are you allowed to do?" The user has already proved who they are (authentication). Now the system checks their permissions. A valid login does not automatically grant access to everything — that would violate the principle of least privilege. Role-Based Access Control (RBAC) is the standard mechanism: the finance database has an ACL that permits only the Finance role. A developer with a valid login is authenticated but not authorized to open the finance database. This separation between authentication and authorization is fundamental to access security.
Q11 SIEM alert: admin login at 03:00 AM from foreign IP, accessed 15 restricted files.
✓ Correct Answer: C — Accounting
The Accounting component of AAA logs session start times, source IP addresses, resources accessed, and data transferred. This is the data that made the SIEM alert possible. Without accounting logs, the SIEM has nothing to correlate. The login event (authentication) and the access check (authorization) are also happening — but the data that paints the full picture of what happened, when, from where, and what was accessed comes from accounting. This is why the accounting layer is called the "security visibility layer" — it feeds directly into the SIEM, and ultimately into analyst investigations.
Q12 Employee denies approving a $200,000 wire transfer; digitally signed approval record exists.
✓ Correct Answer: B — Non-repudiation
Non-repudiation means a party cannot later deny having performed an action. A digital signature uses the signer's private key — only they could have produced it. The corresponding public key can verify it. If the employee's private key signed the approval, they cannot credibly claim they did not approve it. This is why digital signatures matter for high-value transactions. Encryption protects data from being read — it does not prove who performed an action. Non-repudiation is the property that gives logs and digital signatures their legal evidentiary value.
Q13 What does the 3-2-1 backup rule describe?
✓ Correct Answer: B — 3 copies, 2 different media types, 1 stored offline or offsite
The 3-2-1 backup rule is an industry standard for availability protection. Three copies ensures multiple failure points must all fail before data is lost. Two different media types guards against a single media failure (e.g., all your drives failing at once, or all your cloud storage going down). One copy offline or offsite is the critical ransomware defense — ransomware specifically hunts for connected backups to encrypt before triggering on primary systems. An offline backup cannot be reached by malware. Without the offline copy, your backup system can be destroyed in the same attack that targets your primary data.
Q14 Which AAA protocol encrypts the entire communication session?
✓ Correct Answer: C — TACACS+
TACACS+ (Terminal Access Controller Access-Control System Plus) encrypts the entire body of the communication between the client and the AAA server. RADIUS only encrypts the user's password — the rest of the packet, including the username and session metadata, travels in cleartext. TACACS+ also separates the three AAA functions (Authentication, Authorization, Accounting) for more granular control over each. LDAP and Active Directory are directory services used as identity sources — they store user information but are not AAA protocols themselves. TACACS+ is preferred in environments where network device administration requires strong protection of the entire session.
Q15 Average time to contain a credential-theft breach and why it matters to analysts.
✓ Correct Answer: B — 292 days
According to IBM's 2024 Cost of a Data Breach Report, breaches involving stolen credentials took an average of 292 days to identify and contain — the longest of any attack vector. That is nearly 10 months. Every additional day the attacker is undetected is another day they can move laterally, escalate privileges, exfiltrate data, and establish additional footholds. This directly explains why detective controls and fast detection are not optional — they are a financial and operational priority. Breaches contained in under 200 days cost significantly less than those discovered later. The SOC analyst's ability to detect anomalous credential use quickly — unusual login times, foreign IP addresses, abnormal access patterns — is one of the most valuable capabilities in a security program.
